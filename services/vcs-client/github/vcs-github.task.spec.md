# VcsGithubClient

## Цель:

Необходимо реализовать `VcsGithubClient` на базе `services/vcs-client/abstract`.

## Текущие особенности:

Текущий абстрактный класс `VcsClient` по большому чету написан под `services/vcs-client/gitlab` и скорей всего, структура методов и параметров и особенно результатов не гибкая, так gitlab и github имеют разное API как на вход, так и на выход.

Поэтому, первый шагом, нужно унификация InputParams и OutputResults.

## Унификация:

План работы:

1. Изучить ткущий `services/vcs-client/gitlab` составить список метод и их параметры и формат результатов, которые сейчас используются реально используется и сделать чтобы вместо `unknown` создать реальные типы будут описывать возвращаемый результат, например `VcsMergeRequest` или `VcsMergeDiscussion`, важно, нужно именно собрать на основе реальных данных которые используется в проекте.
2. Переименовать все что связанно с `Discussion` в `Thread` как более унифицированное имя (файлы, типа, контракты).
3. Составить списчок текущего API и его аналог из github, главное, указать на разницу и как мы будет устранять разницу, на важно, чтобы на абстрактном уровне интерфейс и параметры имели общий универсальный формат, как и результат, дальше каждая имплементация должна уже переводить их в свой формат.
4. Пишем vcs-github-client и остальные части.

## Важно:

Когда я говорю унификацию, то тебе надо в диалоге с оператором привести план к общему знаменателю и не бояться рефакторинг, если он требуется, что мы могли писать VcsClient под любую систему.
