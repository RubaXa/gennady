# Activity Monitor

Сервис отслеживания любых процессов (ближайшие аналоги MacOS или Chrome Task Manager).

## Основная цель:

Activity Monitor — должен позволять понять:

- Какие процессы запущены (vcs, data ore и так далее).
- Сколько ресурсов потребляют эти процессы, как долго они работаю, не приводят ли они к фризам?
- Как связанны процессы, кт о был инициатором запуска.
- Уничтожение процессов и sub-процессов.
- и так далее (надо проработать)

## Примеры использования:

```ts
// Получаем глобальный монитор
const ac = ActivityMonitor.get();

// Например выполняем запрос к VCS
const prController = new AbortController();
const task = ac.task('code-review', 'vcs.PullRequests', ({ taskSignal }) => {
  // Вызываем метод и передаём сигнал
  return vcs.PullRequests.get(project, iid, {
    signal: taskSignal(prController.signal),
  });
});

// task примерно выглядит так
// {
//   id: number; // ID задачи
//   parentId: number; // ID родительской задачи (0 — нет родителя)
//   initiator: unknown; // Кто запустил, в данном случае 'code-review'
//   descriptor: unknown; // Имя процесса ('vcs.PullRequests')
//   started: number; // hires время запуска
//   duration: number; // getter сколько уже выполняется
//   state: ??; // тут нужны какие-то статусы, чтобы можно было всегда понять, что сейчас происходит с задачей
//   abort: (reason?: unknown) => void; // Отмена отменяет и дочерние процессы
//   promise: Promise<T>;
// }
```

### Открытые вопрос по task:

- Насколько удачное название `task` или всё же `process` или другая производная?
- `initiator` — что это? достаточно ли просто строки или нужно вводить сущность User с правами, как это сделано в OS и подобных системах? Процесс мог запустить система или пользователь или ИИ-агент или езё кто?
- `descriptor` — что это? Просто строка Или может надо передать прямо объект, например `vcs.PullRequests` который должен обладать минимально необходимым набором требований для отображения системной информации.
