## PRIMARY GOAL:
Разработать JavaScript-инструмент для верификации результатов Code Review.
Результатом работы инструмента должен быть:
- `issues` — список найденных проблем в коде.
- `prompt` — промпт, для LLM, на основе которого будет сформирован запрос к LLM для верификации.

## ТРЕБОВАНИЯ:
- Никаких зависимостей, только NodeJs 20 или выше.
- Современный JavaScript.
- Отказоусточивость к ошибкам и исключениям.

## ЛОГИКА РАБОТЫ:

### 1. Пользователь запускает инструмент:

```sh
# Пользователь запускает без указания Merge Request URL:
npx gennady review-verifier # инструмент должен сам определить MR IID из текущего ветки 

# Пользователь запускает с указанием Merge Request URL:
npx gennady review-verifier "<MERGE_REQUEST_URL>"
# или 
# --url="<MERGE_REQUEST_URL>"
# --ref="<PROJECT_ID>!<MERGE_REQUEST_IID>"
# --project="<PROJECT_ID>" --iid="<MERGE_REQUEST_IID>"
```

При запуске, на основе переданных параметров, инструмент должен определить из огружения
- Наличие git
- Из git определить ветку, из которой запущен инструмент
- Из git определить проект, к которому относится MR
- Если не указан Merge Request URL, инструмент должен сам определить MR IID из текущей ветки
- Если в текущей директории нет GIT, и пользователь не указал Merge Request URL, инструмент должен вывести понятную ошибку, что именно не так и повторить попытку.

### 2. Получение информации из GitLab API:
- Проверяем что в `env` есть токен `GITLAB_PERSONAL_TOKEN`, если нет, говорим об этом пользователя, что его нужно добавить в `~/.{zsh|bash}rc` и повторить попытку.
- Загружаем MR и все комментарии к нему из GitLab API

### 3. Формируем результат:

#### `issues`
Сформировать ответ в формате XML, содержащий список найденных проблем в коде (открытых дискуссий).

Пример:
```xml
<MergeRequestReviewReport author="gitlab-login" projectId="<ID>" iid="<IID>" sourceBranch="<BRANCH>" host="<GITLAB_HOST>">
  <Issue discussionId="1" file="path/to/file.ts" line="8">
    <Description><![CDATA[Текст замечания от первичного ревьюера...]]></Description>
    <Note noteId="5" username="gitlab-login"><![CDATA[Дополнительная информация...]]></Note>
  </Issue>
  <!-- ... more issues ... -->
</MergeRequestReviewReport>
```

### `prompt`
Необходимо сформировать промпт, для LLM, на основе которого будет сформирован запрос к LLM для верификации.

Для этого нужно проверить что в проекте есть `<PROJECT>/.ai/ai-review-verifier.xml` (путь должен конфигурироваться в дальнейшем), если нет, то взять  `.ai/ai-review-verifier.xml` из самого `gennady`.

Прочитать этот файл и подставить туда результат как в `issues`.
Те просто склеить `SYSTEM_PROMPT` + `\n` + `issues`.
